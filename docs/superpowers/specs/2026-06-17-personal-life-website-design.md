## Context(为什么 / 做成什么)

用户想要一个**记录自己一生**的个人网站,内容横跨:喜好、音乐、游戏、足迹、许愿树、日记、播客、博客、作品项目等 ~20 个模块。

> **一份内容,两种逛法,四种皮肤。** 内容只写一次(Markdown);展示分"内容模式 / 3D 世界模式"两种可切换形态;外观由统一的主题系统换肤;**架构按"全都要"设计,但分阶段建设**,先把地基与核心模块做扎实,避免重蹈"做太多→推倒"。

## 目标 / 非目标

**目标**
- 一个能长期生长、好维护、纯静态的个人站,内容用 Markdown 维护(用户不写代码)。
- 可扩展架构:加模块=加内容集合;加皮肤=加一组变量;3D 世界作为"模式"后加,与内容站共用数据。
- 第一阶段即可上线、能更新、信息层次清晰。

**非目标(本阶段)**
- 不做后台 / 登录 / 数据库。
- 不做真正的私密内容加密。
- 第一阶段不做 3D 世界、不做播放器、不做小游戏(均为后续增强)。

## 架构

```
顶部切换器:模式(内容 ⇄ 世界) + 主题(4 选 1,存 localStorage)
        │
   ┌────┴─────────────────────────┐
   ▼                              ▼
🖥 内容模式(Astro 多页面,P1)   🌍 3D 世界模式(Three.js 群岛,P2)
   └──────────────┬───────────────┘
                  ▼ 都读取
   📦 共享数据层(Astro Content Collections,类型化)
                  ▲ 来自
   📝 内容源(Markdown / YAML 文件)— 用户唯一编辑处
                  
🎨 主题系统(CSS 变量)包裹整站:换肤=换一组变量,内容页与 3D 世界同时变样
```

- **纯静态**:Astro 默认输出静态 HTML,几乎不发 JS;交互通过"群岛"按需注入(主题切换器、未来的播放器/小游戏/3D 世界)。
- **共享数据层**:内容集合带 frontmatter schema 校验(写错字段即报错,防呆)。内容模式与 3D 世界模式查询同一份集合。
- **可扩展**:`site.config.ts` 用开关控制启用哪些模块——做好一个,打开开关,它才出现在导航与首页。

## 信息架构 / 导航

- **顶部细导航(常驻)**:Logo + 4 个主链接(关于 / 时间轴 / 博客 / 项目)+「更多 ▾」(下拉到全部模块)+ 模式切换 + 主题切换。
- **首页 = 氛围入口枢纽**:一句话自我介绍 + 模块卡片网格(发射台式索引);也是进入 3D 世界的入口。未启用模块灰显"敬请期待"。
- **模块按 6 大主题分组**:关于/创作、喜好、人生记录、愿望/未来、互动/数据、3D 世界。

## 主题系统

- `<html data-theme="...">` + CSS 自定义属性。每个主题 = 一组 token(`--bg / --ink / --accent / --font / --radius / --shadow ...`)。
- 切换器为交互岛,选择存 `localStorage`,刷新保持。
- **第一阶段先上 2 套**:`cozy`(暖粉彩)、`editorial`(极简杂志)——差异大,足以验证换肤机制。`scrapbook`(手帐)、`dark`(暗色赛博)后补。
- 3D 世界模式读取同一组 token(JS 侧读取 CSS 变量映射)。

示例:
```css
[data-theme="cozy"]      { --bg:#fdf3df; --ink:#794f27; --accent:#19c8b9; --font:"Nunito"; --radius:18px; }
[data-theme="editorial"] { --bg:#faf8f4; --ink:#1a1a1a; --accent:#1a1a1a; --font:Georgia,serif; --radius:0; }
```

## 数据模型(内容集合)

```
src/content/
  about.md          # 单条:name, role, bio, avatar, socials[]
  timeline/*.md     # 人生时间轴(主轴)
  blog/*.md         # 博客
  projects/*.md     # 项目
  # 后续:music.yaml, games.yaml, films.yaml, places.yaml, diary/*.md, wishes.md, ...
site.config.ts      # 站名、导航、默认主题、模块启用开关
```

第一阶段各集合的 frontmatter schema:

- **about**(单条):`name`、`role`、`bio`(markdown 正文)、`avatar`、`socials[]{name,url,icon}`
- **timeline 事件**:`date`(必填)、`title`、`type`(life|project|place|music|milestone)、`icon`、`link?`、正文 markdown
- **blog**:`title`、`date`、`tags[]`、`cover?`、`draft`(默认 false)、正文
- **projects**:`title`、`date`、`role?`、`tags[]`、`url?`、`cover?`、`description`、正文

时间轴事件示例:
```markdown
---
date: 2019-09-01
title: 大学入学
type: life
icon: 🎓
link: /projects/xxx   # 可选,关联项目/文章
---
正文 markdown,可含图片。
```

## 项目结构(Astro)

```
src/
  content/        内容(用户只改这里)+ config schema
  themes/         主题变量(cozy / editorial,+后续)
  layouts/        页面骨架(顶部导航 + 切换器)
  components/     卡片 / 时间轴 / 主题切换器 / (后续)播放器、小游戏、3D 世界岛
  pages/          路由:/  /about  /timeline  /blog  /blog/[slug]  /projects  /projects/[slug]
site.config.ts    站点配置 + 模块开关
public/           静态资源(头像、封面图等)
```

## 第一阶段页面设计

- **首页(枢纽)**:细导航条 → 头像 + 一句话介绍 hero → 模块卡片网格(关于我/时间轴/博客/项目 可点;音乐/足迹等灰显"敬请期待")。
- **人生时间轴(主轴)**:竖向时间线,事件含 日期 + 类型彩色圆点 + 图标 + 标题 + 摘要;`project` 类事件可跳转对应项目;顶部可按 `type` 筛选。日后日记/足迹/音乐等均可挂到此轴。
- **博客**:列表页(按日期,过滤草稿)+ 详情页(markdown 渲染、封面、标签)。
- **项目**:列表页(卡片网格)+ 详情页(描述、链接、标签、正文)。
- **关于我**:头像、简介、社交链接;由 `about.md` 驱动。

所有页面套用当前主题;顶部切换器可切 cozy / editorial。

## 模块全集与分阶段路线图

**第一阶段(本次实现计划)**:地基(数据层 + 主题系统 cozy/editorial + 配置开关 + 顶部导航/切换器布局)+ 首页枢纽 + 关于我 + 人生时间轴 + 博客 + 项目 + 部署上线。
